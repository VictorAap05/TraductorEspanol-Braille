# Especificación de Historias de Usuario - Proyecto Braille
**Épica:** Transcribir textos de español a braille incluyendo números, abecedario, vocales acentuadas, y signos básicos.
  
| ID | Historia de Usuario | Criterios de Aceptación |
| :---: | :--- | :--- |
| **HU01** | **Como** usuario<br>**Necesito** que el sistema transcriba correctamente las letras del abecedario (a–z) al Braille<br>**Para** garantizar una conversión precisa. | **Caso de éxito:**<br>- **Dado que** el usuario ingresa texto en minúsculas correspondiente a las letras del abecedario.<br>- **Cuando** el sistema procesa la solicitud.<br>- **Entonces** el sistema debe retornar los puntos correspondientes de la matriz base.<br><br>**Caso de fallo:**<br>- El usuario ingresa texto que incluye caracteres especiales. El sistema falla o devuelve un cuadratín vacío, alterando la señalética. |
| **HU02** | **Como** usuario<br>**Necesito** que el sistema convierta vocales acentuadas (á, é, í, ó, ú) y la ñ a Braille<br>**Para** producir textos completos en español. | **Caso de éxito:**<br>- **Dado que** el usuario ingresa caracteres con acentuación o especiales del español.<br>- **Cuando** el sistema procesa el texto.<br>- **Entonces** el sistema debe retornar la combinación específica para cada una. |
| **HU03** | **Como** usuario<br>**Necesito** que el sistema transcriba números (0–9) al Braille anteponiendo el signo de número<br>**Para** representar cantidades en señalética. | **Caso de éxito:**<br>- **Dado que** el usuario ingresa un número individual.<br>- **Cuando** el sistema genera el código Braille.<br>- **Entonces** el sistema debe anteponer el "signo de número" seguido de la equivalencia de la primera serie primitiva.<br><br>**Caso de fallo:**<br>- El usuario ingresa números combinados con letras, y el sistema no devuelve correctamente el cuadratín señalizando un número. |
| **HU04** | **Como** usuario<br>**Necesito** que el sistema identifique letras mayúsculas y anteponga el indicador de mayúscula Braille<br>**Para** que la transcripción sea fiel al original. | **Caso de éxito:**<br>- **Dado que** el usuario ingresa una letra en mayúscula.<br>- **Cuando** el sistema evalúa el carácter.<br>- **Entonces** el sistema debe añadir un cuadratín previo con el "indicador de mayúsculas".<br><br>**Caso de fallo:**<br>- El usuario ingresa letras mayúsculas, y el sistema no devuelve el indicador de mayúscula braille. |
| **HU05** | **Como** usuario<br>**Necesito** que el sistema me permita generar señalética braille a partir de textos en español<br>**Para** su impresión. | **Caso de éxito:**<br>- **Dado que** el sistema ha procesado con éxito un texto en español.<br>- **Cuando** se despliega el resultado en la pantalla de la aplicación web.<br>- **Entonces** el sistema debe dibujar gráficamente un rectángulo por cada carácter, mostrando los puntos activos rellenos en negro y los inactivos visibles en gris claro.<br><br>**Caso de fallo:**<br>- El usuario solicita la impresión de la señalética, y el sistema devuelve un error o no procesa la vista. |
